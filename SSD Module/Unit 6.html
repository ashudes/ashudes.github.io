Unit 6: Using Linters to Support Python Testing

WHAT?

Week 6 focused on using linters to develop high-quality Python code, underlining the assumption that high-quality code is secure.

SO WHAT?

In the e-Portfolio activity under this Unit, I had a chance to experiment with pylint and flake 8. Unfortunately, I could not run mccabe due to installation issues that I could not figure out. All these tools are useful for code development and testing because they help catch errors and maintain a high standard of code quality. They can catch issues that might be missed during manual code review; and they can enforce a consistent coding style across a project. This can make your code easier to read, understand, and maintain. This understanding is vital for maintaining consistency and error-free code, which are pillars of secure software.

NOW WHAT?

Linters are quite useful for analysing codes, bugs, and style errors. From my experience and some reading, it appears that cannot be relied upon for ensuring code security and functionality. Nonetheless, the use of linters is something I will continue to apply in my future coding projects.
